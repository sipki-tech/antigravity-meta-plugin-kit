# MCP-серверы: mcp_config.json, транспорты, конвенция disabled:true

*[English](mcp.md) | Русский*

MCP-серверы (Model Context Protocol) дают агенту внешние инструменты.
Плагин может привозить собственные серверы; они активируются вместе с
плагином, а их инструменты неймспейсятся при конфликте.

## Конфигурация

Расположения: глобально `~/.gemini/config/mcp_config.json`, в воркспейсе
`<project>/.agents/mcp_config.json` или в плагине
`plugins/<name>/mcp_config.json`.

```json
{
  "mcpServers": {
    "sequential-thinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    },
    "my-local-tool": {
      "command": "my-tool",
      "args": ["mcp"],
      "env": { "MY_TOOL_MODE": "readonly" },
      "disabled": true
    },
    "remote-service": {
      "serverUrl": "https://mcp.mycompany.com/sse"
    }
  }
}
```

Два транспорта:

- **stdio** — `command` (исполняемый файл), `args`, `env`. Сервер
  запускается локально и общается через stdin/stdout.
- **SSE** — `serverUrl` для удалённых серверов по HTTP Server-Sent Events.

## Конвенция `disabled: true`

Если `command` — не универсально доступный лаунчер (`node`, `npx`), у
пользователя без этого бинарника ломаются сессии. Поэтому:

- поставляйте такие записи с `"disabled": true`;
- пусть **инсталлер** обнаруживает бинарник и включает запись только когда
  он реально есть (эталонный antigravity-kit делает ровно так для своего
  опционального CLI `headroom`).

Линт этого кита валит payload, у которого не-встроенная команда идёт без
`disabled: true`. Заметьте: пустой `"mcpServers": {}` валидатор
`agy plugin validate` показывает как `skipped (not found)` — это нормально.

## Неразрушающий merge и prune (контракт инсталлера)

Когда ваш инсталлер мерджит серверы плагина в пользовательский
`mcp_config.json`:

- **никогда не перезаписывайте** запись, которая у пользователя уже есть —
  имя существует → пропускайте;
- при uninstall **удаляйте только записи, байт-в-байт идентичные**
  установленным; всё, что пользователь правил, остаётся.

Скаффолдный `installer/install.mjs` реализует оба правила; скаффолдные
тесты их проверяют (пользовательский сервер переживает install + uninstall).

## Подводные камни

- Обязательный бинарник, поставленный включённым, → сломанные сессии на
  машинах без него.
- Перезапись настроенного пользователем сервера при установке → ярость.
- Секреты в `env`, закоммиченные в репозиторий, — используйте плейсхолдеры
  и документируйте.
- Забыть, что серверы плагина работают только пока плагин включён.

## Чеклист

- [ ] каждая команда не-`node`/`npx` идёт с `disabled: true`
- [ ] инсталлер авто-включает только при обнаруженном бинарнике
- [ ] merge пропускает существующие записи; prune удаляет только точные совпадения
- [ ] нет секретов в закоммиченном `env`
- [ ] `lint` проходит MCP-чеки
