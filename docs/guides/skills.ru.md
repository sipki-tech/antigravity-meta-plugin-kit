# Скиллы: анатомия SKILL.md, триггер-фразы, progressive disclosure

*[English](skills.md) | Русский*

Скилл — это директория со `SKILL.md`: ранбук, который агент загружает **по
требованию**. Antigravity инжектит в контекст только имя + описание
(progressive disclosure); тело загружается при активации скилла. Скиллы
также автоматически становятся /slash-командами.

## Форма директории (официальная)

```
skills/<skill-name>/
├── SKILL.md          # обязателен: frontmatter + инструкции
├── scripts/          # опционально: исполняемые помощники
├── examples/         # опционально: эталонные реализации
├── resources/        # опционально: шаблоны и ассеты
└── references/       # опционально: объёмные доки, читаются по требованию
```

## Frontmatter — поверхность маршрутизации

```markdown
---
name: my-plugin-example
description: Example skill for the my-plugin plugin. Use when the user says "my-plugin-example" or asks what this plugin can do.
---
```

- **`name`** — lowercase-через-дефис, обязан совпадать с именем директории.
- **`description`** — ЕДИНСТВЕННЫЙ текст, который видит роутер. Пишите в
  третьем лице; скажите, **что** скилл делает и **когда** его применять;
  включите триггер-фразу, которую пользователь реально наберёт, лучше в
  кавычках. Описание вида «Database tools» не маршрутизирует никогда;
  «Executes read-only SQL against local PostgreSQL. Use when the user asks
  to inspect the DB» — маршрутизирует.

Линт этого кита проверяет валидность frontmatter и триггер-эвристику
(кавычечная фраза или «use when/for/always/this»).

## Структура тела

Эталонная конвенция, проверенная на двух поколениях плагинов:

```markdown
# skill-name — одна строка, что он делает

## Goal            ← результат, один абзац
## Instructions    ← нумерованные конкретные шаги; команды и пути
## Definition of Done  ← объективно проверяемые пункты
## Constraints     ← чего НЕ делать
## Rationalizations    ← 2–3 пары «отговорка → контраргумент»
```

Нумерованные шаги бьют прозу — агенты надёжно следуют последовательностям.
Включайте шаги самопроверки («проверь лог», «запусти dry-run»), чтобы агент
мог верифицировать себя. Не пересказывайте общеизвестное — модель это знает.

## Токен-гигиена и progressive disclosure

Держите SKILL.md в пределах ~80 строк. Выносите объёмное и линкуйте
относительно:

- длинные справочники → `references/api.md`
- шаблоны промптов → `resources/prompt-template.md`
- многошаговую shell-работу → `scripts/do-thing.sh` (исполняемый, слинкован)

Агент читает слинкованные файлы только при необходимости — в этом и выигрыш
по токенам.

## XML-шаблоны промптов

Сам SKILL.md остаётся markdown. XML-секционирование — рекомендованная Google
форма для длинных многочастных **шаблонов промптов** с плейсхолдерами —
кладите такие в `resources/`:

```xml
<role>…</role>
<constraints>1. …</constraints>
<context>{{USER_REQUEST}}</context>
<task>…</task>
<output_format>…</output_format>
```

Скаффолд несёт рабочий пример
(`skills/<name>-example/resources/prompt-template.md`).

## Портируемость

Тот же формат работает на разных хостах — копируйте директорию скилла в:

| Хост | Расположение |
|---|---|
| Плагин Antigravity | `plugins/<name>/skills/<skill>/` |
| Проект Antigravity | `<project>/.agents/skills/<skill>/` |
| Antigravity глобально | `~/.gemini/config/skills/<skill>/` |
| Claude Code | `~/.claude/skills/<skill>/` |
| Codex | `~/.codex/skills/<skill>/` |

## Подводные камни

- Размытое описание → скилл никогда не срабатывает и молча тратит токены
  payload.
- `name` ≠ имени директории → FAIL линта; путаница маршрутизации.
- Всё заинлайнено в SKILL.md → раздувание контекста в каждой сессии, где он
  активируется.
- Один скилл на три работы → разделите; описание не может маршрутизировать
  трёхголового.

## Чеклист

- [ ] name lowercase-через-дефис и равен директории
- [ ] описание: третье лицо, что + когда, кавычечная триггер-фраза
- [ ] тело ≤ ~80 строк; объёмное — в поддиры, слинковано относительно
- [ ] пункты Definition of Done объективно проверяемы
- [ ] `lint` проходит (frontmatter + триггер-чеки)
