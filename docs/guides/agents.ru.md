# Сабагенты: agents/*.toml

*[English](agents.md) | Русский*

> **Статус формата:** известен валидатору, официально не задокументирован
> (2026-07). `agy plugin validate` считает записи `agents/`, но ни один
> официальный док не описывает поля. Всё ниже — наблюдаемая конвенция из
> работающих плагинов; перепроверяйте после обновлений Antigravity.

Сабагент — специализированная персона, которую привозит ваш плагин: своё
имя, свой системный промпт, своя модель. Главный агент может делегировать
ему подзадачи (эталонный antigravity-kit везёт трёх: планировщика,
архитектора и ревьюера). Живые примеры обоих форматов едут с этим китом:
[plugins/antigravity-meta-plugin-kit/agents/](../../plugins/antigravity-meta-plugin-kit/agents/).

## Форматы файлов

С CLI 1.0.16 засчитываются два формата — TOML (эталонная конвенция) и
markdown с frontmatter (подтверждено пробой, 2026-07-05).

**TOML** — один файл на сабагента в `plugins/<name>/agents/`:

```toml
name = "my-plugin-helper"            # обязан совпадать с именем файла
description = "Read-only помощник: проверяет утверждения по кодовой базе."
nickname_candidates = ["Helper"]     # необязательные алиасы в чате
model = "gemini-3.5-flash"           # выбор модели на агента
developer_instructions = """
Ты — сабагент-помощник плагина my-plugin.
Роль: …
Ограничения: только чтение; никогда не редактируй файлы.
Формат вывода: …
"""
```

Заметки по полям:

- **`name`** — kebab-case, равен имени файла без `.toml`.
- **`description`** — поверхность делегирования: хост инжектит главному
  агенту ростер сабагентов установленных плагинов (имя + описание) с
  формулировкой «you can use them just like regular skills or subagents»
  (подтверждено по всем трём бинарникам Antigravity, 2026-07). Та же
  дисциплина, что у описаний скиллов: что + когда.
- **`model`** — сабагенты не наследуют модель родителя; выбирайте под роль
  (быстрая модель для механических проверок, сильная — для архитектуры).
- **`developer_instructions`** — полный системный промпт. Многострочная
  строка в тройных кавычках; держите пары `"""` сбалансированными.

**Markdown** — форма как у скилла; тело файла — системный промпт:

```markdown
---
name: my-plugin-helper
description: Read-only помощник: проверяет утверждения по кодовой базе.
---
Ты — сабагент-помощник плагина my-plugin. …
```

## Рекомендации по дизайну

- По умолчанию делайте сабагентов **read-only** (исследователь, ревьюер,
  планировщик); давайте право записи только когда работа этого требует — и
  проговаривайте это в промпте.
- Одна работа на сабагента. «Планировщик, который заодно чинит линт» плохо
  делегируется.
- Явно задавайте формат вывода в `developer_instructions` — его потребляет
  оркестратор.

## Скаффолд и валидация

```bash
# opt-in: формат недокументирован, поэтому за флагом
npx github:sipki-tech/antigravity-meta-plugin-kit create my-plugin --with-agents
```

`lint` этого кита проверяет каждый `agents/*.toml` (когда папка есть):
`name` присутствует и совпадает с файлом, `description` присутствует, пары
`"""` сбалансированы — строчные эвристики, не TOML-парсер. Официальный
валидатор их считает: `agy plugin validate` → `agents: N processed`. На CLI
1.1.1+ `agy agents` показывает агентов, доступных сессиям, а
`agy --agent <name>` выбирает одного — самая быстрая живая проверка, что
ваш агент зарегистрировался.

## Подводные камни

- `name` ≠ имени файла → FAIL линта; делегирование может промахнуться.
- Несбалансированные `"""` → весь файл не парсится.
- Промпт без формата вывода → оркестратор получает прозу, которую не может
  потребить.

## Чеклист

- [ ] один TOML на сабагента, name == имени файла
- [ ] description говорит что + когда (поверхность делегирования)

- [ ] developer_instructions: роль, ограничения, формат вывода
